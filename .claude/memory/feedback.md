# 運用ルール

## プロジェクト特有のものは絶対にプロジェクト内に置く（Git管理前提）

プロジェクトに属するもの（メモリー・ルール・計画・ノート・設定・ドキュメント等）は、必ずそのプロジェクトのディレクトリ内にGit追跡可能な状態で配置する。グローバル・ホーム・ルート等のGit範囲外の場所には絶対に置かない。

逆に、複数プロジェクトに跨る汎用的・横断的なドキュメントは、特定プロジェクト内に置かず、Git管理対象外の場所（`~/.claude/` 等）で管理する。プロジェクトのGitリポジトリは「そのプロジェクトに属するものだけ」を含む状態に保つ。

**Why:** プロジェクト固有の情報が Git 管理外 (`~/.claude/projects/<proj>/memory/` や `~/.claude/plans/` 等) に堆積すると、共有もバックアップも効かない異物になる。逆方向に汎用ルールを特定プロジェクトに混入させると、そのリポジトリにとって異物になる。

**How to apply:** 何かを保存する前に「これはプロジェクト特有か、それとも汎用か」を必ず先に判定する。

置いてはいけない組み合わせ:
- プロジェクト特有のものを `~/.claude/memory/` に置く
- プロジェクト特有のものを `~/.claude/projects/<...>/memory/` に置く（auto memory、Git共有外）
- プロジェクト特有のものを `~/.claude/plans/` に置く
- プロジェクト特有のものを `~/.claude/CLAUDE.md` に書く
- プロジェクト特有のものを `~/.zshrc` 等のホーム配下dotfileに書く
- 他プロジェクトの情報を本プロジェクトの中に書く
